# MVP — UX Polish (Приоритет 3)

Доводка пользовательского опыта после реализации ядра.

---

## P3.1 — QR-код в лобби

- Генерация через qrcode.js (CDN)
- Кодирует URL: `https://<username>.github.io/bdtrivia/?room=CODE`
- Показывается крупно на экране хоста (min 200×200px)
- Перегенерируется при изменении комнаты

---

## P3.2 — Короткий код комнаты

- 4-6 символов, отображается крупным шрифтом под QR-кодом
- Игроки могут ввести вручную на странице входа (если нет камеры)

---

## P3.3 — Нумерация по раундам

Игрок видит:

```
Раунд 1: История — Вопрос 2 / 5
```

Логика:
- `roundName` берётся из последнего пройденного slide (или из текущего вопроса)
- Счётчик вопросов сбрасывается при каждом новом slide-разделителе
- Items с типом `slide` не учитываются в нумерации

---

## P3.4 — Счётчик ответивших (для хоста)

Хост видит во время вопроса:

```
Ответили: 3 / 8
```

- Обновляется в реальном времени через Firebase `on('value')`
- Показывает количество записей в `answers/<currentItem>/`

---

## P3.5 — Промежуточный топ-3

После каждого вопроса (в фазе `reveal`):

- Все игроки видят краткий топ-3 по текущему счёту
- Формат: `🥇 Ник — 3400  🥈 Ник — 2800  🥉 Ник — 2100`
- Для хоста: полный список вместо топ-3 (с прокруткой)

---

## P3.6 — Экран «Ждём остальных»

После отправки ответа игрок видит:

- «Ответ принят! Ждём остальных...»
- Время до конца таймера (мелко)
- Анимация ожидания (пульсирующий индикатор или точки)

---

## P3.7 — Обработка отключений

- При `players/<id>/connected === false` (или при `disconnect`): игрок помечается как отключившийся
- Его ответы за текущий вопрос сохраняются
- Дальнейшие вопросы: ответ считается пропущенным (0 баллов)
- Игра продолжается без блокировок

---

## P3.8 — Новая игра

Кнопка на финальном экране:

1. Подтверждение: «Начать новую игру? Текущие результаты будут потеряны»
2. Удаление `/rooms/<code>/` из Firebase
3. Очистка `localStorage`
4. Перезагрузка страницы (возврат в режим редактора)
