# MVP — Question Editor (Приоритет 2)

Редактор вопросов на странице хоста (перед созданием комнаты).

---

## P2.1 — Тип «выбор» (choice)

Форма добавления вопроса:

- **Тип:** селектор → «Выбор одного»
- **Вопрос:** textarea (обязательно)
- **Варианты:** 2-6 полей ввода, кнопки «+ Вариант» (до 6), «×» (минимум 2)
- **Правильный ответ:** радио-кнопка рядом с вариантом

Валидация: все поля непустые, правильный выбран.

---

## P2.2 — Тип «свободный текст» (text)

Форма добавления вопроса:

- **Тип:** селектор → «Свободный текст»
- **Вопрос:** textarea (обязательно)
- **Правильный ответ/синонимы:** одно поле, синонимы через запятую

Валидация: вопрос непустой, хотя бы один синоним указан.

Проверка ответа: `correctAnswer.some(s => s.toLowerCase().trim() === playerAnswer.toLowerCase().trim())`

---

## P2.3 — Кастомный maxScore

Числовое поле на вопрос, по умолчанию 1000.

- Ограничение: 100–10000
- Шаг: 100

---

## P2.4 — Кастомный timeLimit

Числовое поле на вопрос, по умолчанию 20.

- Ограничение: 5–120 секунд
- Шаг: 5

---

## P2.5 — Медиа

Поле URL + селектор типа:

- **Типы:** YouTube / Аудио / Картинка
- **YouTube:** ссылка на видео → извлекается ID → вставляется iframe-превью
- **Аудио:** прямая ссылка на mp3/ogg → HTML5 audio-плеер в превью
- **Картинка:** прямая ссылка на изображение → img-превью

Поле необязательное. Если пусто — медиа нет.

---

## P2.6 — Слайды-разделители

Кнопка «+ Разделитель раунда» добавляет элемент с типом `slide`:

- **Заголовок:** text (обязательно), например «Раунд 1: История»
- **Описание:** textarea (опционально), например правила раунда

Слайд вставляется в общий список и отображается при игре как полноэкранный разделитель.

---

## P2.7 — Reorder

Кнопки ↑↓ на каждом элементе (вопросы и слайды):

- ↑ — сдвинуть на одну позицию вверх
- ↓ — сдвинуть на одну позицию вниз
- Границы: первый элемент не может ↑, последний не может ↓

---

## P2.8 — Удаление

Кнопка «× Удалить» на каждом элементе:

- Подтверждение: «Удалить вопрос/слайд?»
- После удаления индексы пересчитываются

---

## P2.9 — Сохранение черновика

- Автосохранение в `localStorage.setItem('bdtrivia_draft', JSON.stringify(items))` при каждом изменении (debounce 1-2 сек)
- При загрузке страницы: проверка `localStorage.getItem('bdtrivia_draft')`, предложение восстановить
- Кнопка «Очистить» сбрасывает черновик
