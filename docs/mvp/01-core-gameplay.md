# MVP — Core Gameplay (Приоритет 1)

Ядро игры: комната, подключение, полный игровой цикл.

---

## P1.1 — Комната + Firebase

Хост нажимает «Создать комнату»:

1. Генерируется короткий код (4-6 символов: A-Z + цифры, без 0/O/1/I/L)
2. Создаётся узел `/rooms/<code>/` в Firebase RTDB
3. Все последующие операции читают/пишут в этот узел

**Зависимости:** Firebase SDK (через CDN)

---

## P1.2 — Подключение игрока

Игрок открывает `index.html?room=CODE`:

1. Вводит никнейм (обязательно, непустой)
2. Нажимает «Присоединиться»
3. Пишет свои данные в `/rooms/<code>/players/<localId>/` (nickname, score: 0, connected: true)
4. Подписывается на `/rooms/<code>/` для получения обновлений
5. Хост видит нового игрока в реальном времени через `on('child_added')`

---

## P1.3 — Старт игры

Хост нажимает «Начать игру»:

1. `room.state = "playing"`
2. `room.currentItem = 0`
3. Если первый item — slide, показывается хостом, ждёт «Далее»
4. Если первый item — question, запускается таймер

---

## P1.4 — Показ вопроса (choice)

Игрок видит:

- Текст вопроса
- Медиа (если есть): YouTube iframe / аудиоплеер / картинка
- N вариантов ответа (радио-кнопки или кнопки на весь экран)
- Таймер обратного отсчёта
- Прогресс-номер: «Раунд N → Вопрос X / Y»

Хост видит то же самое + счётчик ответивших.

---

## P1.5 — Показ вопроса (text)

Игрок видит:

- Текст вопроса
- Медиа (если есть)
- Однострочное поле ввода + кнопка «Ответить»
- Таймер обратного отсчёта
- Прогресс-номер

**Валидация:** case-insensitive match с любым из синонимов в `correctAnswer[]`.

---

## P1.6 — Слайды-разделители раундов

При `items[currentItem].type === "slide"`:

1. Игроки видят: заголовок + описание (крупно, по центру), без таймера и ответов
2. Хост видит: заголовок + описание + кнопка «Далее»
3. Хост нажимает «Далее» → `currentItem++`
4. `timerEndsAt` = null (пока не дойдём до вопроса)

---

## P1.7 — Приём ответов

Игрок выбирает ответ → нажимает «Ответить»:

1. Запись в `/rooms/<code>/answers/<currentItem>/<playerId>/`:
   - `answer`: строка ответа
   - `answeredAt`: server.Time (или Date.now())
2. Игрок видит экран «Ждём остальных...»
3. Повторная отправка игнорируется (проверка наличия ключа)

---

## P1.8 — Таймер

При переходе на вопрос:

1. `timerEndsAt = Date.now() + timeLimit * 1000`
2. Все клиенты синхронно отсчитывают время
3. По истечении: приём ответов закрывается, неответившие получают 0 баллов
4. Если все ответили раньше — таймер всё равно идёт до конца (минимум 5 секунд? или до конца — решить)

---

## P1.9 — Подсчёт очков

При окончании таймера (или при ответе):

```
elapsed = answeredAt - currentItemStartTime
ratio   = elapsed / (timeLimit * 1000)
score   = maxScore × max(0, 1 − ratio × 0.85)

score = 0 если ответ неверный или отсутствует
```

Формула даёт:
- Мгновенный правильный ответ → `maxScore` (1000 по умолч.)
- Ответ на последней секунде → ~`0.15 × maxScore` (150)
- Неверный / пустой → 0

---

## P1.10 — Раскрытие результата

После таймера `state = "reveal"`:

- Игрок видит: правильный ответ, свой ответ, начисленные баллы
- Хост видит: правильный ответ, распределение ответов (гистограмма для choice, % правильных для text)
- Все видят промежуточный топ-3

---

## P1.11 — Переключение между item'ами

Хост нажимает «Далее»:

1. `currentItem++`
2. Если новый item — slide: показать, ждать «Далее»
3. Если новый item — question: запустить таймер, начать приём
4. Если `currentItem >= items.length`: финальная таблица

---

## P1.12 — Финальная таблица

После последнего item'а `state = "finished"`:

- Подиум: 1-е, 2-е, 3-е место (крупно)
- Полная таблица всех игроков со счётом (сортировка по убыванию)
- Кнопка «Новая игра» (сброс Firebase + localStorage)

---

## P1.13 — Сериализация/десериализация

При перезагрузке хоста:

1. Чтение `state.json` из localStorage (или из URL? из Firebase по коду комнаты)
2. Восстановление состояния игры (текущий вопрос, ответы, таймер)
3. Если игра была в процессе — продолжить с того же места
